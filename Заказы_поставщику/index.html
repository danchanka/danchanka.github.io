Заказы поставщику

Чтобы использовать EDI для согласования заказов с поставщиками, в первую очередь необходимо заполнить данные по поставщику и товарам и произвести некоторые настройки, о чем подробно написано в статье  Настройки EDI. 

Алгоритм согласования заказа​

**1. **В системе формируется заказ на закупку в соответствии с существующими алгоритмами. 

2. Чтобы отправить заказ поставщику для согласования, на форме Закупки - Закупки - Заказы на вкладке Действия используем кнопку Отправить заказы поставщикам.  Система отправит все отмеченные заказы или, если заказы не отмечены, выделенный курсором заказ.  В поле Доставка будет отображаться метод отправки. Если у поставщика указан EDI провайдер, то будет отображаться EDI: имя_провайдера (в других случаях отправка заказа осуществляется по электронной почте, если указан e-mail). 

Когда в систему поступит сообщение от провайдера, подтверждающее, что  поставщик получил заказ (рис. 3),  его статус измениться на **Отправлен **и будет отмечен чекбокс Отправлен. 

  Рис. 1 Отправка заказов на согласование​

3. При получении ответа поставщика по заказу, он переходит в статус Подтвержден и в заказе заполняются поля Подтвержденное количество (EDI) и **Подтвержденная цена (EDI) **(рис. 2). 

  • Если поставщик подтверждает запрошенные вами количество и цену, то заказ Проводят и ожидают поставку.
  • Если поставщик подтверждает количество и цену, отличные от заказанных, то поля подсвечиваются красным, а статус заказа изменяется на С изменениями (рис. 1).
    • Покупатель может согласится с условиями заказчика, изменить количество и цену, провести заказ и получить поставку по новым условиям. 

или 

    • Если покупатель не согласен с условиями поставщика, можно Отменить заказ. На форме Закупки - Закупки - Заказы на вкладке Действия использовать кнопку Отменить заказ (рис. 1). Если для организации  поставщика указан e-mail, то информация об отмене заказа будет также отправлена на электронную почту.

Рис. 2 Подтверждение заказа​

Сообщения по заказам EDI​

Все заказы на закупку, обмен которыми происходит посредством EDI, можно просмотреть на форме Финансы - EDI - Заказы EDI. По умолчанию отображаются открытые заказы, которые можно отправить как электронный документ. Отправку можно осуществить непосредственно на данной форме выделив заказ курсором и нажав Отправить.  

Отмеченные чекбоксы указывают на статус заказа:

  • Экспортирован -  заказ отправлен поставщику
  • Экспортирован (отмена) - заказ отменен
  • Возврат - это  заказ закупка-возврат

  Рис. 3 форма Заказы EDI​

Состав заказа отображается на вкладке Строка заказа EDI, а сообщения по заказам на вкладке Сообщение по заказу EDI. На этой вкладке можно просмотреть сообщение, нажав кнопку Открыть xml. 

Ответы поставщика по заказам отображаются на форме Финансы - EDI - Ответы по **Заказам EDI. **Просмотреть электронное сообщение-ответ поставщика можно нажав кнопку **Открыть xml **в строке ответа. В нижней части формы указаны данные, которые прислал поставщик в ответ на ваш заказ. 

Рис. 4 Форма Ответы по Заказам EDI​

Накладная по уведомлению об отгрузке​

Когда заказ согласован с поставщиком, то при отправке груза поставщик может прислать уведомление об отгрузке, в котором указаны заказанное и фактически отгруженное количество и согласованная цена. Полученные уведомления об отгрузке можно просмотреть на форме Финансы - EDI - Уведомления об отгрузке EDI. Само сообщение уведомления можно просмотреть нажав на кнопку  Открыть xml в строке уведомления. На базе полученного уведомления можно Создать накладную одноименной кнопкой.  Все необходимые для накладной данные будут заполнены. 

Рис. 5 Форма Уведомление об отгрузке EDI​

Также при создании накладной по заказу на форме Приемка по заказам, система будет предлагать использовать поступившие уведомления об отгрузке, если такие есть. Необходимо нажать Создать накладную, и выбрать уведомление о доставке, в соответствии с которым будет создана накладная.
